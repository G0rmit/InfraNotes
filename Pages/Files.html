File Management

home

File system is an important part of OS, that helps locate and manage files, organize data with folders and track file meta data.

All data is converted to binary 1 and 0. Text characters each have their own code.

Sounds are more complicated, the encoding is done via "sampling". Microphones measure how much the air is displaced by the sound with diaphragm. However, there is a limit to how often samples can be done, which is called "sampling rate"

Images

There are two types of images:

  • Vector image

    Uses mathematical description of shapes, hence lines

  • Raster image

    Uses pixels, that have colors thanks to RGB values(24 bits in each pixel).

Metadata

Metadata is a data about data. It is divided in header, main ad footer. Header contains file type and metadata: size, date, author...

An important part of metadata is a "magic number". It is 8-digit code at the beginning of the file code that specifies file type.

How File System Works?

  • When creating file:

    Assigns space on disk and records file metadata.

  • When opening file:

    Reads metadata to find file's location and loads file content into memory.

  • When deleting file:

    Marks disk space available andkeeps metadata for a short time.

File systems in different OS

Windows uses NTFS and fat32(older), macOS uses APFS and HFS+APFS, Linux uses ext4, Btrfs and XFS.

File Compression

There are two types of compression: lossless(zip, png, gif, flac) and lossy(mp3, jpeg, mpeg). Lossy compression permanently deletes some pixels that are not visible to human eye in order to save more space.

There are different compresson algorithms. One of them is "Redundancy reduction" or "Run-Length Encoding", and another one is "Dictionnary based methods"

Redundancy reduction replaces consecutive repeated values with a single value and a count. Example: aaaaabbb turns into 5a3b.

Dictionary methods scans for repeated sequenses and stores each unique sequence in a dictionary with a short code.
